Lists & Tuples

A list is an ordered, mutable collection. A tuple is ordered but immutable — use it for data that shouldn't change.

Python 
# Lists — ordered and mutable
fruits = ['apple', 'banana', 'cherry']
fruits.append('date')
fruits[0] = 'avocado'
print(fruits)       # ['avocado', 'banana', 'cherry', 'date']
print(fruits[1:3])  # ['banana', 'cherry']

# Tuples — ordered but immutable
coords = (10.5, 20.3)
x, y = coords
print(f'x={x}, y={y}')

Dictionaries

A dictionary stores key-value pairs. Keys must be unique and immutable. Extremely fast for lookups.

Python 
person = {
    'name': 'Alice',
    'age': 25,
    'city': 'Paris'
}
print(person['name'])           # Alice
person['email'] = 'a@b.com'

for key, val in person.items():
    print(f'{key}: {val}')

Sets & Comprehensions

A set stores unique, unordered values. Comprehensions are a concise, Pythonic way to build collections.

Python 
# Sets — unique values only
nums = {1, 2, 2, 3, 3, 4}
print(nums)        # {1, 2, 3, 4}

# List comprehension
squares = [x**2 for x in range(1, 6)]
print(squares)    # [1, 4, 9, 16, 25]

# Dict comprehension
word_len = {w: len(w) for w in ['hello', 'world', 'python']}
print(word_len)
