File Handling

Work with real-world data stored in files. Learn to read and write text, CSV, and JSON files safely using Python's context managers and exception handling.

⏱ ~30 minutes 📝 4 sections 🎯 1 quiz

Reading & Writing Text Files

Use open() with a with statement — it automatically closes the file even if an error occurs.

Python 
# Writing — 'w' creates or overwrites
with open('notes.txt', 'w') as f:
    f.write('Python is awesome!\n')
    f.write('File handling is easy.\n')

# Reading the whole file
with open('notes.txt', 'r') as f:
    content = f.read()
    print(content)

# Line-by-line iteration
with open('notes.txt', 'r') as f:
    for i, line in enumerate(f, start=1):
        print(f'Line {i}: {line.strip()}')
Mode Description
'r'Read (default) — file must exist
'w'Write — creates or overwrites
'a'Append — adds to end of file
'x'Exclusive create — fails if file exists

Working with JSON

JSON is the de facto format for APIs and configuration. Python's json module handles serialization and deserialization effortlessly.

Python 
import json

data = {
    'name':   'Alice',
    'age':    25,
    'skills': ['Python', 'SQL', 'Git']
}

# Serialize Python → JSON
json_str = json.dumps(data, indent=2)
print(json_str)

# Deserialize JSON → Python
parsed = json.loads(json_str)
print(parsed['skills'])

Exception Handling

Always wrap file operations in try/except to gracefully handle errors like missing files.

Python 
try:
    with open('missing.txt', 'r') as f:
        data = f.read()
except FileNotFoundError:
    print('File not found!')
except PermissionError:
    print('Permission denied!')
except Exception as e:
    print(f'Unexpected error: {e}')
finally:
    print('Cleanup complete.')
